## 2. 社区需求归类

从 50 条 Issue 中归纳出以下尚未被满足的工作流方向（并非预测趋势，仅记录已提出的需求）：

- **组织级分享与管理**：Issue #228（13 条评论）要求能在组织内直接共享 Skill，无需手动下载上传。目前只能通过文件传递，缺少共享库或链接。
- **安全与信任边界**：Issue #492（6 条评论）指出社区 Skill 以 `anthropic/` 命名空间分发，可能误导用户授予过高权限，要求建立官方认证机制。
- **重复与负载问题**：Issue #189（6 条评论）报告 `document-skills` 和 `example-skills` 插件安装后内容重复，浪费上下文；Issue #1087（2 条评论）补充 `document-skills` 加载了所有 Skill 而非仅声明的 4 个。
- **评估与测试工具缺陷**：Issue #556（8 条评论）指出 `run_eval.py` 运行测试时所有查询都未能触发 Skill 调用（0% 触发率），影响 Skill 质量验证。
- **企业身份认证兼容**：Issue #532（已关闭，2 条评论）提到 skill-creator 的描述优化脚本依赖 `ANTHROPIC_API_KEY`，但企业 SSO 用户无法使用。
- **Agent 治理模式**：Issue #412（已关闭，4 条评论）提议新增 `agent-governance` Skill，包含策略执行、威胁检测、信任评分等内容，但未获合并。
- **平台拓展**：Issue #29（4 条评论）询问如何在 AWS Bedrock 上使用 Skill；Issue #16（4 条评论）要求将 Skill 暴露为 MCP 接口。
- **数据量控制**：Issue #1102（2 条评论）反馈 MCP 返回数据未压缩，大量数据导致上下文拥塞。

### 2. 社区需求归类

从 Issues 中可以观察到社区正在关注以下几类未满足的需求：

- **技能分发与管理**：
  - 用户强烈要求组织级别的技能共享功能（#228），并反映了当前手动分享 `.skill` 文件流程的低效。
  - 因仓库结构问题（`document-skills` 和 `example-skills` 插件内容重复）导致的重复加载问题被多次提及（#189, #1087）。

- **平台兼容性与稳定性**：
  - 官方评估工具 `run_eval.py` 在 Windows 平台上存在严重问题，导致技能触发率为 0%（#556, #1099）。
  - 用户报告了 Skills 在特定客户端（如 Claude.ai 和 Claude Desktop）上加载失败，或在列表显示后消失的 Bug（#61, #62）。

- **安全性**：
  - 社区对通过 `anthropic/` 命名空间分发的社区技能表达了信任边界担忧，认为这可能误导用户授予不安全的权限（#492）。
  - 用户在设计涉及 SharePoint Online 文档处理的系统时，提出了将访问控制逻辑直接写在 `SKILL.md` 文件中的安全隐患（#1175）。

- **工作流集成与新方向提案**：
  - 有用户提出向 AI 代理系统添加治理模式的“agent-governance”Skill 提案（#412）。
  - 用户希望 Skills 能被暴露为 MCP 工具，以统一 AI 软件的接口协议（#16）。
  - 存在使用 AWS Bedrock 运行 Skills 的兼容性需求（#29）。

### 2. 社区需求归类 (从 Issues)

从 Issue 中可以归类出社区下列未满足的需求方向：

- **组织级协作与分发：** 用户强烈希望能在组织内部直接共享 Skill，而不是通过下载文件手动上传。 (`#228`)
- **安全与信任边界：** 社区对社区Skill的治理提出质疑，认为分布在 `anthropic/` 命名空间下的社区技能存在信任风险，可能导致用户误判并授予不必要的权限。 (`#492`)
- **基础设施集成与优化：**
    - 希望 Skill 能通过 MCP (Model Context Protocol) 暴露为可调用的 API。 (`#16`)
    - 希望能在 AWS Bedrock 等其他平台上使用 Skills。 (`#29`)
- **平台稳定性与数据一致性：**
    - 存在技能无故消失的错误报告。 (`#62`)
    - 插件安装时存在加载重复技能 (`#189`) 和加载了未声明的技能 (`#1087`) 的问题。
- **特定领域扩展：**
    - 社区提出了对“AI Agent 治理”技能的需求，涉及策略执行、威胁检测和审计跟踪。 (`#412`)

### 2. 社区需求归类 (Issues)

从当前采样的 Issues 中，社区未被满足的需求主要集中在以下方面：

- **平台与基础设施**:
    - **组织级共享**：用户需要能够在组织内直接共享 Skills，而非通过手动下载、传输文件并上传的繁琐流程 (#228)。
    - **MCP 集成与数据优化**：用户提出 Skill 应能通过 MCP 暴露 API，并关注数据返回量过大导致上下文拥堵的优化问题 (#16, #1102)。
    - **云平台兼容性**：明确提出了 Skills 与 AWS Bedrock 等平台的集成需求 (#29)。

- **技能内容与质量**:
    - **Agent 安全治理**：有提案希望新增“agent-governance”技能，以解决 AI agent 系统中的策略执行、威胁检测和审计追踪等安全问题。这反映了对技能安全性的进阶需求 (#412)。
    - **多文件引用与打包**：用户反馈当一个 Skill 依赖多个参考文件时，当前仅加载 `SKILL.md` 的机制无法满足需求，提出了多文件预加载或内联打包的特性请求 (#1220)。

- **工具稳定性与使用体验**:
    - **数据与功能异常**：用户报告了技能文件消失、加载时出现404错误、以及两个插件安装后产生内容重复和冲突等问题，反映出工具链的健壮性需要加强 (#62, #61, #189)。
    - **工具名称混淆**：社区技能与官方技能在同一命名空间下，可能导致用户混淆，带来安全风险 (#492)。

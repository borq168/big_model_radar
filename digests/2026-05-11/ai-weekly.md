## 2. CLI 工具进展

本周，AI CLI 工具的核心矛盾从“功能创新”转向“**稳定与安全**”。

-   **Claude Code**：整体负面消息较多。除了计费与 RCE 漏洞，其在 WSL 下的 Agent Teams 重复实例、Opus 4.6/4.7 模型指令忽略等问题也持续困扰用户。正面动作为发布了紧急补丁（v2.1.131-137）修复部分问题，并引入了 `hard_deny` 安全规则。
-   **OpenAI Codex**：社区活跃度极高（单日新增400+ Issue），核心诉求为**远程控制**（最高赞）和 Windows 独立安装。版本更新频繁（Rust v0.129.0 ~ v0.131.0），但 Token 消耗异常飙升问题用户反馈强烈。
-   **Gemini CLI**：持续高强度迭代，修复了子代理误报、Shell 超时等多个问题，并集成 ACP 协议。社区主要关注代理行为异常和内存泄漏问题。
-   **GitHub Copilot CLI**：重点修复了 macOS/Windows 下的静默崩溃和非交互模式兼容性问题（v1.0.44），并新增 `userPromptSubmitted` 钩子，增强可观测性。但其 Agent 在任务中途静默停止的问题（回归）依然严峻。
-   **Kimi Code CLI**：一个重要动向是将默认 Shell 从 PowerShell 切换为 **Git Bash**（PR #2186），以根本解决 POSIX 命令兼容性问题。Windows 兼容性是其核心痛点。
-   **Qwen Code**：发布了 v0.15.9 稳定版和多个 Nightly 版，引入了文件读取缓存。但 `write_file` 等工具的文本误识别为二进制 Bug 严重影响使用。免费层 OAuth 额度调整引发社区激烈争论。
-   **OpenCode**：发布 v1.14.34-48 等多个版本，主要修复 TUI 配置模式冲突、图像附件兼容性问题，但 `/exit` 命令消失等小问题也时有发生。

**共性趋势**：
1.  **会话管理与恢复**：几乎所有工具都在解决会话丢失、子代理状态不一致的问题。
2.  **跨平台兼容性**：Windows/WSL 是主要短板，各工具都在积极适配。
3.  **MCP (Model Context Protocol) 生态**：MCP 服务器模式、懒加载等需求成为社区高频请求，是工具间互联互通的关键。

## 3. AI Agent 生态

-   **OpenClaw**：作为开源 Agent 框架的代表，本周处于**高度活跃与快速迭代**状态。
    -   **版本发布**：连发 v2026.5.4 至 v2026.5.10-beta 等多个版本，Sprint 速度极快。
    -   **主要功能**：新增基于 Telegram 的 QA 自动化测试、`/think default` 命令、实时语音桥接（Google Meet/Twilio）、以及核心的 **memory-core log-memory** 子系统（支持混合检索和梦境循环）。
    -   **严重问题**：稳定性问题突出，包括内存泄漏、会话文件无限增长、Subagent 结果静默丢失等。`doctor --fix` 功能曾因破坏性写入导致用户环境锁定，引发信任危机。
    -   **社区需求**：强烈渴望桌面客户端（Linux/Windows）和移动端（Android APK）支持，以及“掩码密钥”等安全功能。
-   **同赛道项目**：NanoBot、Zeroclaw、PicoClaw 等信息较少，但“Git for AI Agents”（regent-vcs/re_gent）项目出现在 HN 讨论中，为 Agent 工作流管理提供版本控制思路，值得关注。

**核心矛盾**：OpenClaw 等开源 Agent 框架正处在“**功能大爆发**”与“**稳定性严重滞后**”的矛盾期，快速迭代带来了大量 Bug，对早期采用者的使用体验是一大考验。

### 2. 社区需求归类

从近期的 Issues 中，可以观察到社区对以下 Skill 方向的需求：

- **组织级协作与共享**: `#228` 提出 Skills 应在组织内直接共享，而非通过下载再上传的繁琐方式。这反映了团队化使用场景的增长。
- **评估与测试工具链**: `#556` 报告 `run_eval.py` 脚本无法正确触发 Skills，暴露出社区对技能评估和测试工具稳定性的依赖与不满。`#202` 和 `#532` 则分别指出了官方 `skill-creator` 技能本身存在的设计问题和可用性缺陷（如依赖 Anthropic API Key）。
- **安全与信任边界**: `#492` 指出社区技能发布在官方 `anthropic/` 命名空间下可能误导用户信任，并形成安全漏洞。这反映了社区对技能来源认证和权限隔离的敏感性提升。
- **系统集成与互操作性**: `#16` 提议将 Skills 作为 MCP（Model Context Protocol）服务器暴露，`#29` 则询问与 AWS Bedrock 的兼容性。这表明社区希望将 Skills 能力集成到更广泛的 AI 工具链中，而非仅局限于 Claude 生态。
- **特定领域深度技能**: `#412` 提议了“智能体治理”（agent-governance）这一新技能方向，聚焦于 AI 代理系统的安全模式、策略执行和审计追踪。这暗示了 AI 应用开发正在从“能否实现功能”向“如何安全可控地实现”演进。

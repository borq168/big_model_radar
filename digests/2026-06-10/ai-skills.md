### 1. 热门 Skills 排行

以下按近期社区关注度（评论数、讨论热度）筛选出 5 个代表性的 Skills PR：

1.  **文档排版质量控制** (`document-typography`, PR #514)
    - **功能**: 让 Claude 在生成文档时避免“孤字成行”、“段首孤行”、“标题悬垂”等常见排版问题。
    - **讨论热点**: 社区认为这是 AI 文档生成的“最后一公里”痛点，但具体实现逻辑（如如何精确定义和修正糟糕的换行）仍在讨论中。
    - **状态**: OPEN，持续活跃。
    - 链接: https://github.com/anthropics/skills/pull/514

2.  **OpenDocument 格式支持** (`odt`, PR #486)
    - **功能**: 支持创建、读取和转换 `.odt` 和 `.ods` 格式文件，提供与 LibreOffice 等开源办公套件的互操作能力。
    - **讨论热点**: 社区对 ISO 标准格式的支持有明确需求，但实现细节（如模板填充、ODT 到 HTML 的转换鲁棒性）是讨论焦点。
    - **状态**: OPEN，讨论中。
    - 链接: https://github.com/anthropics/skills/pull/486

3.  **前端设计技能优化** (PR #210)
    - **功能**: 对现有 `frontend-design` 技能进行重构，目标是让指令更清晰、可操作，确保 Claude 能与用户在同一次对话中协作完成前端设计。
    - **讨论热点**: 重点在于如何将抽象的设计原则转化为 Claude 能够一步步执行的精确指令，提升技能的有效性和一致性。
    - **状态**: OPEN，已收到较多反馈。
    - 链接: https://github.com/anthropics/skills/pull/210

4.  **元技能：技能质量与安全分析器** (PR #83)
    - **功能**: 提议新增两个元技能，用于自动评估其他 Skills 的质量（结构、示例、文档等）和安全性（权限、风险等）。
    - **讨论热点**: 评审者关注评估标准的客观性与覆盖面，以及如何避免偏向贡献者的主观偏好。

### 2. 社区需求归类

从 Issues 中可以归类出以下几类社区关注方向：

- **平台与功能扩展需求**
    - **组织级共享** (Issue #228)：多个用户期望能在组织内直接共享 Skills，而非通过文件手动传输。
    - **AWS Bedrock 兼容** (Issue #29)：用户询问如何在 AWS Bedrock 环境下使用这些 Skills。
    - **MCP 协议暴露** (Issue #16)：提议将 Skills 封装为标准 MCP 服务，以便更广泛地集成。
    - **多文件捆绑** (Issue #1220)：建议支持将多个参考文件（如 `refs/*.md`）打包，解决当前仅 `SKILL.md` 被传递，导致引用丢失的问题。

- **工具链与开发体验**
    - **评估工具缺陷** (Issue #556, #1169)：`run_eval.py` 和优化循环在特定场景下（如检查 slash-commands）出现 0% 触发率/召回率，导致工具无法正常工作。**这是当前社区反馈最集中、最核心的开发工具链问题**。
    - **安全与信任** (Issue #492)：社区治理问题显现，有人担忧非官方 Skills 伪装成 `anthropic/` 命名空间下的官方技能，构成信任滥用风险。
    - **可移植性标准** (Issue #1156)：社区讨论如何为不同的 Skills 标注其可移植性（是通用还是仅适用于特定项目），以降低用户的使用心智负担。

- **新技能方向提议**
    - **Agent 治理** (Agent Governance, Issue #412)：社区提议新增一个指导 AI Agent 系统进行策略执行、威胁检测和审计的技能，填补当前安全治理方向的空白。
